# MixTV 首页模块产品说明

本文档说明首页的模块组成、展示规则、数据来源和后台配置方式，方便产品、运营、研发和测试统一理解 MixTV 的首页能力。

## 1. 首页定位

首页是用户进入站点后的核心发现页，承担以下目标：

1. 快速展示当前可看的热门内容。
2. 提供继续观看入口，帮助用户无缝回到上次进度。
3. 展示即将上映和番剧日历等时效内容。
4. 支持后台按模块开关，适配不同运营策略。

## 2. 首页内容结构

首页内容分为两类：

1. 固定展示区域
2. 可配置模块区域

### 2.1 固定展示区域

这些内容默认都会参与展示，其中欢迎横幅可由首页配置单独控制显示与否：

1. **加载态遮罩**：首页初次进入时展示 Cinematic Loading 动画。
2. **欢迎横幅**：顶部问候语和站点视觉横幅，展示用户昵称和欢迎语，受 `showWelcomeBanner` 控制。
3. **Telegram 欢迎弹窗**：面向新用户的引导弹窗。
4. **公告弹窗**：如果站点配置了公告且用户未看过，则显示一次。

### 2.2 可配置模块区域

首页模块的默认展示顺序如下：

1. Hero Banner 轮播
2. 继续观看
3. 即将上映
4. 热门电影
5. 热门剧集
6. 新番放送
7. 热门综艺
8. 热门短剧

## 3. 模块说明

| 模块 | 面向用户的价值 | 展示条件 | 数据来源 | 关联入口 |
| --- | --- | --- | --- | --- |
| Hero Banner 轮播 | 首屏推荐重点内容，增强首页视觉聚焦 | `showHeroBanner = true` 且有可用数据 | 由首页热门数据组装，取前 2 部电影、前 2 部剧集、前 1 部综艺、前 1 部动漫，并补充详情中的封面、背景图、预告片和简介 | 无固定列表入口 |
| 继续观看 | 让用户快速回到上次播放进度 | `showContinueWatching = true` | 播放记录、收藏和继续观看相关查询结果 | 继续观看组件内交互 |
| 即将上映 | 展示近期即将上线或已上映的影片 | `showUpcomingReleases = true` 且有数据 | `/api/release-calendar?limit=100`，再通过 `releaseCalendar.worker.ts` 处理筛选 | `查看更多` 跳转 `/release-calendar` |
| 热门电影 | 展示热门电影 | `showHotMovies = true` | 豆瓣热门电影数据 | `查看更多` 跳转 `/douban?type=movie` |
| 热门剧集 | 展示热门电视剧 | `showHotTvShows = true` | 豆瓣热门剧集数据 | `查看更多` 跳转 `/douban?type=tv` |
| 新番放送 | 展示当前日期对应的番剧条目 | `showNewAnime = true` 且有数据 | 番剧日历数据 + 番剧聚合数据 | `查看更多` 跳转 `/douban?type=anime` |
| 热门综艺 | 展示热门综艺 | `showHotVariety = true` | 豆瓣热门综艺数据 | `查看更多` 跳转 `/douban?type=show` |
| 热门短剧 | 展示短剧推荐 | `showHotShortDramas = true` | 短剧推荐数据 | `查看更多` 跳转 `/shortdrama` |

## 4. 首页展示规则

1. 模块是否渲染，先由后台配置控制。
2. 模块开启但数据为空时，不展示空白模块。
3. 首页数据在客户端并行拉取，关闭的模块不会触发对应查询，减少无效请求。
4. 首次加载时，页面优先显示加载态，避免半成品内容逐块闪现。
5. 热门短剧如果检测到空缓存，会主动触发一次刷新，避免长期显示空数据。
6. Hero Banner 依赖首页热门内容和详情补充数据，若没有可用条目则不显示。

## 5. 后台配置说明

后台在“首页模块配置”中提供 9 个开关，对应首页的 8 个可配置模块和 1 个欢迎横幅开关：

1. `showHeroBanner`
2. `showWelcomeBanner`
3. `showContinueWatching`
4. `showUpcomingReleases`
5. `showHotMovies`
6. `showHotTvShows`
7. `showNewAnime`
8. `showHotVariety`
9. `showHotShortDramas`

### 配置行为

1. 保存接口：`POST /api/admin/homepage-config`
2. 读取接口：`GET /api/admin/homepage-config`
3. 保存后会刷新首页缓存，并触发首页路径重校验。
4. 默认值全部为 `true`，也就是首次安装后完整首页默认开启。

### 约束说明

1. 本地存储模式不支持管理员配置首页模块。
2. 配置保存依赖管理员身份登录。

## 6. 数据来源与依赖

首页依赖的数据类型主要有以下几类：

1. **热门内容数据**：用于热门电影、剧集、综艺、动漫，以及 Hero Banner 的基础内容。
2. **详情增强数据**：用于补充封面、背景图、预告片、剧情简介等增强信息。
3. **番剧日历数据**：用于新番放送模块和当天番剧筛选。
4. **即将上映日历数据**：用于即将上映模块。
5. **短剧推荐数据**：用于热门短剧模块。
6. **用户播放相关数据**：用于继续观看模块。
7. **站点公告配置**：用于公告弹窗展示。

## 7. 异常和降级

1. 某个模块请求失败时，不影响其他模块的展示。
2. 首页会保留已有数据并继续渲染，避免因为局部失败导致整页不可用。
3. 即将上映模块在 Worker 不可用时会降级为空列表。
4. 当首页数据首次加载中，卡片区域使用占位骨架屏。
5. 用户已看过公告时，公告弹窗不会重复打扰。

## 8. 适用场景

这份说明适合用于：

1. 产品讨论首页模块展示策略。
2. 运营调整首页开关前的影响确认。
3. 测试验证首页模块是否按配置生效。
4. 研发排查首页数据缺失、模块不显示或配置未生效的问题。
